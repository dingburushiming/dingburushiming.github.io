Git和GitHub的区别

Git:世界上最先进的分布式版本控制系统，是一个用于记录一个或若干文件内容变化的软件

GitHub:一个存放文件，提供git服务的网站，有订阅，关注，讨论组，在线编辑器等功能

Git的安装配置

1.安装好后需要设置用户名，已经用户邮箱

$ git config –global user.name

$ git config –global user.email

2.创建目录仓库

mkdir resp_name

3.进入该目录

cd /resp_name

4.初始化该仓库,执行之后会在项目目录下创建一个.git隐藏目录

git init

git的相关指令

查看当前状态

git status

添加到缓存区

git add 文件名

git add 文件1 文件2 文件3…

git add . 【添加当前目录到缓存区】

提交到版本库

git commit -m “注释内容”

版本回退

步骤一：查看版本

git log

git log –pretty=oneline

步骤二：回退指令

git reset –hard 提交编号

注意回滚之后，回滚编号之后的版本号就不会显示，想要查看所有版本号要通过指令

git reflog

git连接github管理项目的两种常规方式

基于http协议

拉取项目(进入仓库文件夹进行)

git clone 线上仓库地址

提交暂存区

git add .

提交本地仓库

git commit -m 注释

提交线上仓库

git push

如果提交上线仓库时出现了403权限问题可以在.git文件里面加上账号密码信息

拉取线上仓库

git pull

每天工作的第一件事情就是git pull拉取项目更新

每天下班最后一件事情就是git push提交代码

基于ssh协议

与https相比只是影响github对于用户的身份鉴权方式，对具体操作无影响

创建公私钥对文件

$ ssh-keygen -t rsa -C “2234191776@qq.com“

在对应的位置找到公钥，填入github对应的页面

分支管理

一个模块就是一个分支，分支彼此之间不影响，更加安全

相关指令

查看分支：git branch

创建分支：git branch 分支名

切换分支：git checkout -d 分支名【有-d表示创建切换】

删除分支：git branch -d 分支名

合并分支：git merge 被合并的分支名

注意切换分支后的数据会发生相应的变化，但是都是在同一个本地文件中

删除分支只能删除被合并的分支，不能删除合并后的分支;且删除分支时，该分支不能处于使用状态

忽略文件

忽略原因

对于不想提交的文件，采用忽略文件机制来实现需求，需要新建一个名为.gitignore的文件

注意：.gitignore文件没办法直接在windows目录下创建，可以通过命令行git bash来touch创建

创建.gitignore文件

touch .gitignore

忽略用法

最前面加上！就是方向不过滤的意思

/mtk/ 过滤整个文件夹

*.zip 过滤所有的.zip文件

/mtk/do.c 过滤某个具体文件夹
